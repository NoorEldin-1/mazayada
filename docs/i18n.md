# Internationalization (i18n) Guide

Mazayada is a **trilingual** platform: **Arabic (`ar`, primary + fallback)**,
**French (`fr`)**, and **English (`en`)**. Arabic is RTL; French and English are
LTR. Translating every user-facing string to all three languages is the
**default**, not an afterthought — a test enforces it.

---

## The rules (read this first)

1. **Never hardcode user-facing text.** Use `__('group.key')` in Blade and PHP.
2. **Every key exists in all three locales** — `lang/ar`, `lang/fr`, `lang/en`.
   The `LocalizationTest` fails CI if a key is missing in any locale.
3. **Arabic is the source of truth.** Write the `ar` value first, then `fr`/`en`.
4. **Reference data** (Category, Wilaya, Commune, Entity) uses the localized
   `->name` accessor — never read `->name_ar` directly in new code.
5. **Direction-aware CSS.** Use logical properties (`margin-inline-start`,
   `inset-inline-end`, `text-align: start/end`) — not `left`/`right` — so the
   layout mirrors automatically between RTL and LTR.

---

## Where things live

| Concern | Location |
|---|---|
| Supported locales, dir, fonts, labels | `config/locales.php` |
| Translation strings | `lang/{ar,fr,en}/<group>.php` |
| Locale resolution (per request) | `app/Http/Middleware/SetLocale.php` |
| Language switch endpoint | `route('lang.switch', $locale)` → `routes/web.php` |
| Switcher UI | `<x-lang-switcher />` (`resources/views/components/lang-switcher.blade.php`) |
| User's saved preference | `users.locale` column + `User::preferredLocale()` |
| Direction / font / lang helpers | `app/Helpers/helpers.php` (`locale_dir()`, `locale_lang()`, `locale_is_rtl()`, `locale_font_class()`) |
| Reference-data localized name | `App\Models\Concerns\HasLocalizedName` |
| Key-parity guardrail | `tests/Feature/LocalizationTest.php` |

## Translation groups (files)

`common`, `nav`, `footer`, `enums`, `auth`, `home`, `auctions`, `kyc`,
`dashboard`, `validation`. Add a new group file when a feature introduces a
distinct cluster of strings (e.g. `appeals.php`, `payments.php`). Create it in
**all three** locale folders.

---

## How locale is resolved (priority)

`SetLocale` runs on every web request and picks the locale in this order:

1. `?lang=` query parameter (an explicit switch this request)
2. `session('locale')` (what the visitor last chose)
3. authenticated `user->locale` (follows them across devices)
4. `config('locales.default')` → `ar`

The result is stored back in the session, and `Carbon::setLocale()` is aligned
so relative dates render in the active language. A guest's choice on the landing
page is carried into their account at registration (`AuthController@register`).

## Recipes

### Add a string to an existing screen
```php
// lang/ar/dashboard.php
'welcome_back' => 'مرحباً بعودتك',
// lang/fr/dashboard.php
'welcome_back' => 'Bon retour',
// lang/en/dashboard.php
'welcome_back' => 'Welcome back',
```
```blade
<h1>{{ __('dashboard.welcome_back') }}</h1>
```

### Placeholders
```php
'greeting' => 'مرحباً :name',           // ar
__('common.greeting', ['name' => $user->name]);
```

### A string that contains a link (HTML)
```blade
{!! __('auth.terms_agree', [
    'terms'   => '<a href="#">'.__('auth.terms_link').'</a>',
    'privacy' => '<a href="#">'.__('auth.privacy_link').'</a>',
]) !!}
```

### Enum labels
Enum `label()` methods return `__('enums.<group>.<CASE>')`. Add the case to
`lang/{ar,fr,en}/enums.php`.

### Reference-data names
```blade
{{ $category->name }}   {{-- localized; do NOT use $category->name_ar --}}
{{ $auction->wilaya->name }}
```

### Money
`dzd($centimes)` / `dzd_short($centimes)` already localize the currency suffix
(`دج` / `DA`) via `common.currency`.

## Layouts

Every layout's root element is direction-aware:
```blade
<html lang="{{ locale_lang() }}" dir="{{ locale_dir() }}">
```
Include `<x-lang-switcher />` in the header of any new layout.

---

## Before you push

```bash
php artisan test --filter=LocalizationTest
```
If it fails, it prints exactly which keys are missing in which locale. Fix them
— do not skip the test.

## Scope / roadmap

- **Done (foundation):** UI strings, validation, enums, reference-data names,
  user preference, working switcher, RTL/LTR.
- **Fully converted:** every Blade view (landing, auth, all citizen pages, all
  admin pages, public auctions, static pages, pagination) plus all backend
  user-facing strings (controller flash/errors, enum `label()`s, custom Rules,
  the language switcher in all four layouts including admin).
- **Future feature:** multilingual auction free-text (admin enters title /
  description per language). The schema and accessor pattern already support it
  — auction `title_ar` / `description_ar` are still rendered as-is for now.
